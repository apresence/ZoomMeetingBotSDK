This project depends on the Zoom Meeting SDK C# Wrapper.  Details here:
https://marketplace.zoom.us/docs/sdk/native-sdks/windows/c-sharp-wrapper/

Note that Zoom enforces minimum SDK versions, so the C# Wrapper must be updated from time to time.  Details here:
https://marketplace.zoom.us/docs/sdk/overview/minimum-version/

There are often breaking changes in the API. Details here:
https://devsupport.zoom.us/hc/en-us/articles/9564556977933-How-to-update-your-SDKs-to-newer-versions

Setup:

- Download the latest version of the SDK and extract it somewhere. In my dev environment, I extract it to a 3rdparty folder. Ex: 3rdparty\zoom-c-sharp-wrapper-5.12.2.9319

- Open the README.md file from the extracted folder and review the additional requirements there

- Open x86\zoom_sdk_csharp_wrap\zoom_sdk_c_sharp_wrap.sln in Visual Studio 2022

  - Retarget Projects:
    - Windows SDK Version: Pick whichever ends with "(latest installed version)"
    - Platform Toolset: Pick the Upgrade option. For example, "Upgrade to v143", but this may differ on your system

  - Change the project's solution configuration to "Release" and platform to "x86"

  - Right-click on zoom_sdk_dotnet_wrap, then select Build > Rebuild

- UsherBot uses some features of the SDK that are not included in the wrapper by default.  In order to add these features, code modifications to the following .cpp/.h files are necessary:

  git diff -p --no-index -- x86\zoom_sdk_c_sharp_wrap\%1 x64\zoom_sdk_c_sharp_wrap\%1 > %~nx1.patch
  git apply --unsafe-paths meeting_participants_ctrl_donet_wrap.cpp.patch
